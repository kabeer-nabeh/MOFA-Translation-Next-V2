# Product

## Register

product

## Users

Two overlapping groups:

- **MOFA diplomatic staff** (primary hosts): Ministry of Foreign Affairs officials managing and running multilingual meetings. They operate in formal, high-stakes professional contexts where precision and reliability are non-negotiable.
- **International delegates** (guests): External participants from foreign delegations. They join meetings expecting seamless real-time translation with no learning curve.

Both groups use the product during or around live meetings. The workflow is time-sensitive: a mistranslation or UI failure has real diplomatic consequences.

## Product Purpose

Baseer Voice is a live simultaneous translation platform built for MOFA. It enables Arabic/English (and other language) translation in real-time across in-person and video meetings, with recording, transcript review, and analytics. Success means diplomats can run meetings as if the language barrier doesn't exist.

## Brand Personality

Calm, precise, trusted. Like a skilled simultaneous interpreter: invisible when things go right, indispensable when they matter. The UI should feel like a professional tool that respects the gravity of diplomatic work without being heavy or bureaucratic.

Three words: **reliable, refined, unobtrusive**.

## Anti-references

- **Heavy enterprise (SAP/Oracle)**: Dense, grey-on-grey, utilitarian. Makes users feel like they're filing expense reports.
- **Overly branded government tools**: Heavy institutional crests, flag colors, formal-but-ugly. Design choices that announce "government software" before the user has done anything.

## Design Principles

1. **Invisible when it works**: The interface should recede. When translation is flowing, the UI should feel like clear glass, not a product asserting itself.
2. **Diplomatic weight without visual noise**: Formal without being heavy. Precision typography and restrained color over ornamentation.
3. **Language as a first-class citizen**: Arabic and English are co-equal. RTL layout, Arabic typography, and bidirectional content must feel native, not bolted-on.
4. **Clarity at speed**: Users act during live meetings. Every interaction must be decisively obvious, zero ambiguity, minimal cognitive load.
5. **Trust through consistency**: Inconsistency (mismatched colors, unpredictable spacing, unclear states) erodes confidence in a tool that handles high-stakes communication.

## Accessibility & Inclusion

- Target: WCAG 2.1 AA
- Arabic/RTL support is a core requirement, not an afterthought
- Reduced motion preference should be respected (live translation animations are prominent)
- Color contrast must hold for both LTR and RTL rendering contexts
